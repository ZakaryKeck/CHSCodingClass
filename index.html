Concurrent Metronomes

You may have seen this type of video on TikTok, and today we're going to be recreating it! All the necessary instructions are bolded if you get lost or stuck.

Step 1 - Drawing a Circle

Open this Online IDE we will be using: https://openprocessing.org/. Click that link, and the sketch that opens has some starter code for you. 

function setup() {
  createCanvas(windowWidth, windowHeight);
  background(100);
}

function draw() {
  circle(mouseX, mouseY, 20);
}

setup and draw are two "functions" like we talked about in the powerpoint. They are unique because they are provided for us. setup is run 1 time right after you click play, and draw is run 60 times every second!

Click the play button at the top. A new circle will be drawn over your mouse 60 times each second. mouseX and mouseY are two more variables provided for us. They represent the x-coordinate and y-coordinate of your mouse, respectively. Click the </> icon next to the play button to go back to the code.

createCanvas, background, and circle are three more functions. They create the area where we can draw, color the background, and draw a circle respectively.

Notice how the circle doesn't go away when you move your mouse. What do you think would happen if we move the background(100) out of setup and into draw? 

function setup() {
  createCanvas(windowWidth, windowHeight);
}

function draw() {
  background(100);
  circle(mouseX, mouseY, 20);
}

Next, draw some other circles in various locations, and with various sizes. The parameters for the circle function are (x-coordinate, y-coordinate, circleDiameter).

Step 2 - Circular Motion

When you're making games and animations, you can think of the x axis on a graph as time. As we move forward in time, x increases. frameCount is another variable provided to us that tells us how long the program has been running - it starts at 0 and increases with every frame. Now replace the circle function in your code with this line:

circle(frameCount, 0, 20);

Next, we're going to use a little trigonometry to make the circle move in a circle. Here is an image that illustrates the anatomy of a wave. I like to use https://www.geogebra.org/calculator to visualize functions. Let's visualize sin and cos there.

Next, replace the circle function with this line:

circle(cos(frameCount), 0, 20);

You should be able to see the circle wiggling in the top left corner. How can we increase the size of the circle's movement? Increase the amplitude of the wave times 100!

circle(100*cos(frameCount), 0, 20);

Now the circle is speeding around the top of the canvas. Keep in mind that (0,0) on the canvas is the top left corner - x increases as you move right, and y increases as you move down. So how can we move it into the center? Add vertical and horizontal shift!

circle(100*cos(frameCount) + (width/2), height/2, 20);

Next, we want to slow the circle down. To do that we increase the period of the wave.

circle(100*cos(frameCount/30) + (width/2), height/2, 20);

Lastly, how can we use our knowledge of the two types of waves to create circular motion? Take a look at the sin and cos graphs again. Notice that when cos(x) = 1, sin(x) = 0. We can use this to give us the point (1,0). If you continue through one full period of the waves, you can see the points (1, 0), (0, 1), (-1, 0), (0,-1). The waves compliment each other to create circular motion! Replace your code with this line:

circle(100*cos(frameCount/30) + (width/2), 100*sin (frameCount/30) + (height/2), 20);

Step 3 - Starter Code

Open up this Sketch which has starter code: https://openprocessing.org/sketch/1543865. Much of the code that you need already exists, but it is "commented out". This means that there are two forward slashes // in front of it. A comment can also start in the middle of a line. When we say "uncomment something", just erase those two slashes.

At the top of the Sketch, we have all our variables for the program. Then we have an Entity class. Then we have the setup and draw functions from before. Lastly, we have some useful functions. Click play, and you see the v shape from the video, but it's not in the middle. Let's quickly go over where it needs to be. Afterwards, replace the points x1 through y3 with the commented out points in the variables at the top.

Next, let's look at the Entity class. It has 10 properties, which is another word for variables, and 5 functions. Let's talk about how we can use the trigonometry from before to bounce the entities back and forth. Afterwards, replace the return statements with the commented out code for getNewX and getNewY. Remember to either erase or comment out the return statements below the new lines.

Incredible! One ball is bouncing! Next can you find a line to uncomment that looks like it would play the sound we want?

Next, uncomment a line that would give the entity the correct color (blue). Let's add some more colors from this list https://www.w3.org/wiki/CSS/Properties/color/keywords. They need to go in the colors list at the top, with the variables.

Now, let's take a look at the for loop that creates the entities. The variable that determines how many entities there are is called numberOfEntities. Change it to add some more.
